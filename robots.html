Differential ground robot Simulator

Develop a differential robot simulator with following scenarios :

  • A target moves in a defined environment and is plotted,
  • A robot follows that target, but avoid walls (obstacles) along the way while following the target,
  • The first robot embeds simulated sensors, and a second robot is instanciated,
  • The second robot is now following the path of the first robot according to its sensor actuators.
  • Running the current algorithm plots the positions of each robot and the target for a defined simulation time,
  • The object-oriented approach allows to easily instanciate multiple robots and sensors.

Technologies Used: C++/Python

Project proposed by: Olivier Kermorgant

STAGE Simulation - Maze Solving Robot

Simulation of the Maze Solving Robot for maze completion (exploration mode) :

  • Development of a ROS package that uses STAGE as a Simulation Environment,
  • Creation of different kind of maps with more or less tricky situations,
  • Test of different algorithms starting by the left-hand/right-hand rule algorithm,
  • Final decision was leaning towards a left-hand/right-hand rule mixed up with a PID controller to smoothen the angles.

Technologies Used: C++, ROS, Control Theory

Other Contributor(s): Nicolas Filliol, Jan Tiepelt, Oleksander Koreiba, Giovanni Alexander Bergamaschi

Pathfinder - Maze Solving Robot

ROS package featuring :

  • Maze Solving with the A* Algorithm taking an occupancy grid as input,
  • For the demo, the robot previously maps the maze with a LiDaR during the right-hand/left-hand rule explorer,
  • Starting point and target point are set with the previous exercise,
  • The robot then follows the found shortest path.

Technologies Used: C++, Python, ROS, Pathfinding

Other Contributor(s): Nicolas Filliol
